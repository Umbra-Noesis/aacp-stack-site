The Problem

Prompting is guesswork. AACP is structure.

A prompt is an unstructured natural language request to a system that doesn't know what you mean — only what you said. The system guesses at intent, guesses at context, guesses at what tool to use. AACP eliminates that guesswork by defining the contract both sides of the conversation actually follow.

Every AACP message carries structured fields: who is sending, what context is active, what the intent is, what rules apply, and what the expected response form is. Nothing is inferred. Nothing is ambiguous. The communication is deterministic by design.

Structured message contracts — not free-form natural language

Context binding — session, task, and environment travel with every message

Policy enforcement at the protocol level — not bolted on afterward

Auditable — every exchange follows a known schema, traceable after the fact

Layer Breakdown

AACP Core

The base schema that defines every message field — sender, intent, context reference, tool binding, and response contract. Everything else extends from this layer.

Context & State Layer

Tracks the active environment, current task, and session state so every response is grounded in what is actually happening — not just what was last said.

UN-SLP Symbol Layer

Lightweight symbol tags for communication mode, tone, and intent that sit above AACP Core. Enables fast, low-overhead modifiers without restructuring the base message.

Tool & Runner Binding

Encodes exactly which tools, runners, or workflows should execute for a given message. No ambiguity about what should happen — the protocol carries the instruction.
